## 1. Вход в панель администратора

1. Открыть `/admin` на сайте
2. Ввести логин и пароль (по умолчанию: `admin` / `changeme`)
3. **Сменить пароль** при первом входе: вкладка **Settings → Change Password**

> **Важно:** Если на экране жёлтый баннер «You are using the default password» — пароль ещё не сменён. Это критично для безопасности.

## 2. Создание опроса

### Шаг 1 — Создать новый опрос

1. Открыть вкладку **Surveys**
2. Нажать **New Survey**
3. Ввести название (например: `Опрос — весна 2026`) и описание
4. Нажать **Create**

### Шаг 2 — Добавить вопросы

Нажать **Edit** на опросе, затем перейти в раздел вопросов:

| Поле | Что указывать |
|------|--------------|
| Category | `academic` / `sleep` / `emotional` / `social` |
| Question Type | `scale_1_5` — шкала от 1 до 5; `choice` — выбор из вариантов |
| Scale Low/High | Подписи к крайним значениям шкалы (например: `Never` / `Always`) |
| Inverted | Включить, если высокий ответ означает *хорошо* (например: «Как вы оцениваете своё самочувствие?») |

**Для вопросов типа `choice`:**

- Варианты ответов вводятся через запятую
- Для каждого варианта указывается `scoring_map` — числа от 0 до 100, где 100 = максимальное выгорание

Пример scoring_map для вопроса о сне:

```json
{"<4h": 100, "4-5h": 80, "5-6h": 60, "6-7h": 30, "7-8h": 10, "8+h": 0}
```

### Шаг 3 — Активировать опрос

Нажать **Activate** на опросе. Только один опрос может быть активным — именно он показывается пользователям.

> **Примечание:** Не удаляйте опросы с ответами — система не позволит, но лучше просто деактивировать.

### Рекомендуемая структура опроса

```
3 вопроса — Academic (академическая нагрузка)
3 вопроса — Sleep (сон и энергия)
3 вопроса — Emotional (эмоциональное состояние)
3 вопроса — Social (социальная жизнь)
```

Минимум 2 вопроса на категорию для статистически значимых результатов.

## 3. Понимание шкалы выгорания

Все ответы нормализуются в шкалу **0–100**, где:

| Диапазон | Уровень | Цвет |
|----------|---------|------|
| 0–20 | Low (норма) | зелёный |
| 21–40 | Mild (лёгкое) | синий |
| 41–60 | Moderate (умеренное) | оранжевый |
| 61–80 | High (высокое) | красный |
| 81–100 | Critical (критическое) | ярко-красный |

**Итоговый скор** = среднее по четырём категориям.

**Инвертированные вопросы:** вопрос «Насколько вы довольны своей учёбой?» — высокий ответ (5) означает хорошее самочувствие. Система автоматически инвертирует: ответ 5 → скор 0%.

## 4. Раздел Analytics — обзор

Открыть вкладку **Analytics**. Верхние карточки показывают сводку по текущим фильтрам:

- **Total Responses** — общее число заполненных опросов
- **Avg Burnout Score** — средний суммарный скор (0–100%)
- **Critical Cases** — количество ответов со скором >80
- **Distribution bar** — мини-диаграмма распределения по уровням

---

## 5. Фильтры

Панель фильтров под карточками применяется **ко всем разделам** аналитики одновременно.

| Фильтр | Описание |
|--------|----------|
| From / To | Диапазон дат. Пресеты: Today, Last 7d, Last 30d, All time |
| Course Years | Фильтр по курсу (1–5) |
| Department | Фильтр по институту |
| Period | Период семестра |
| Exclude suspicious | По умолчанию включён — исключает подозрительные ответы |

**Совет:** При анализе данных сначала просматривайте картину «All time», а затем сужайте до конкретного периода (например, только «Exam period») для сравнения нагрузки.

## 6. Summary — разбор по вопросам

Раздел **Summary** показывает распределение ответов отдельно по каждому вопросу.

### Вопросы типа шкала (scale_1_5)

Горизонтальные цветные полосы показывают, сколько % студентов выбрали каждое значение (1–5).

```
1 ████ 8%     (Never)
2 ██████ 15%
3 ████████████ 32%
4 ██████████ 28%
5 ██████ 17%  (Always)

Mean: 3.31 | n=146
```

- **Mean** (среднее) — ключевое число. Выше 3.5 — тревожный сигнал по этому вопросу.
- Цвет полос: от зелёного (1) до красного (5).

### Вопросы с выбором (choice)

Горизонтальные бары для каждого варианта. Самый популярный вариант подсвечен оранжевым.

**Как читать:**

- Если большинство выбирают крайние «плохие» варианты (например, сон <4–5h) — это системная проблема
- Смотрите в контексте: для сна «6–7h» считается нормой у студентов

## 7. Categories — профиль выгорания

Раздел **Categories** показывает:

### Score Bars

Четыре полосы для каждой категории. Цвет соответствует уровню выгорания.

**Как интерпретировать:**

- **0–40%** — норма, внимание не требуется
- **40–60%** — умеренная нагрузка, рекомендуется мониторинг
- **60–80%** — высокий уровень, нужны интервенции
- **80%+** — критично, срочные меры

**Типичная картина:** Academic и Emotional обычно выше, чем Sleep и Social. Если Sleep критически высокий — это часто маркер общего неблагополучия.

### Radar Chart (паутинная диаграмма)

Визуализирует «форму» выгорания. Идеальная картина — маленький многоугольник близко к центру. Выраженные «шипы» в одной категории указывают на специфическую проблему.

**Паттерны:**

- Равномерно большой многоугольник → общее хроническое выгорание
- Шип в Academic → перегруз учёбой
- Шип в Emotional → стресс/тревога
- Шип в Sleep → хроническое недосыпание

## 8. Cross Analysis — сравнительный анализ

Три режима (переключаются кнопками):

### By Course (по курсам)

Сравнивает выгорание студентов разных курсов по всем 4 категориям.

**Типичные паттерны:**

- 1 курс: высокий Emotional (адаптация), невысокий Academic
- 3–4 курс: пиковый Academic (дипломные, сессии)
- 5 курс: снижение по всем метрикам (или резкий рост Academic)

Сгруппированная гистограмма + таблица с тепловой раскраской ячеек:

- Строки — курсы, столбцы — категории
- Более красные ячейки = выше выгорание в этой категории на этом курсе

### By Department (по институтам)

Определяет, какие институты испытывают наибольшую нагрузку.

> Таблица отсортирована по суммарному скору — самые «нагруженные» институты вверху.

### By Period (по периоду семестра)

Сравнивает выгорание в разные периоды учебного года.

**Ожидаемая картина:**

```
Beginning < Mid-semester < Exam period > Between semesters
```

Если Exam period значительно выше (>15 пунктов от Beginning) — это нормально. Если выгорание уже высокое в начале — системная проблема.

## 9. Timeline — динамика во времени

Два графика:

### Response Volume (количество ответов по времени)

Показывает, когда студенты активнее всего заполняют опрос. Пики = активное распространение или акция.

### Score Trend (тренд скора)

5 линий: Total (суммарный) + 4 категории.

**Как читать:**

- Восходящий тренд в периоды сессий — нормально
- Если тренд растёт даже в начале семестра — тревожный сигнал
- Расхождение категорий (например, Academic растёт, Sleep нет) — специфическая нагрузка

**Совет:** Используйте фильтр дат «Last 30d» для оперативного мониторинга. «All time» — для понимания долгосрочных трендов.

## 10. Demographics — кто проходит опрос

| Чарт | Что показывает |
|------|----------------|
| By Course Year | Pie-диаграмма распределения по курсам |
| By Semester Period | Когда студенты проходят опрос |
| By Department (top 8) | Горизонтальный бар — самые активные институты |

**Зачем это нужно:**

- Если 80% ответов с 1–2 курса, результаты нерепрезентативны для старших курсов
- Дисбаланс по институтам → нужно дополнительное продвижение в недопредставленных
- Пики активности → привязка к событиям (анонс, рассылка)

## 11. Responses — отдельные ответы

Таблица всех индивидуальных ответов с сортировкой.

### Сортировка

Клик по заголовку столбца — сортировка по возрастанию/убыванию. Полезные сортировки:

- По **Total** (убывание) → найти самые тяжёлые случаи
- По **Date** (убывание) → последние ответы
- По **Course Year** → группировка по курсам

### View Details

Клик **View** → модальное окно с полным ответом:

- Все ответы вопрос за вопросом
- Нормализованный скор по каждому ответу
- Итоговые скоры по категориям
- Время прохождения

**Флаг ⚠** — ответ помечен как подозрительный (прошёл за <15 секунд). Такие ответы по умолчанию исключены из статистики.

### Удаление ответов

Кнопка **×** удаляет ответ безвозвратно. Перед массовой очисткой сделайте бэкап.

## 12. Экспорт данных

Кнопки в правом верхнем углу раздела Analytics:

### Export Raw CSV

Все ответы построчно. Столбцы:

```
id, created_at, course_year, department, semester_period,
score_academic, score_sleep, score_emotional, score_social, score_total,
ip_hash, fingerprint, is_suspicious, completion_time_seconds
```

### Export Summary CSV

Агрегация по комбинации department × course_year. Столбцы:

```
department, course_year, count, avg_total, avg_academic, avg_sleep, avg_emotional, avg_social
```

**Для работы в Excel / Google Sheets:**

1. Скачать CSV
2. Открыть через «Данные → Из текста/CSV» (кодировка UTF-8)
3. Использовать сводные таблицы для детального анализа

## 13. Подозрительные ответы

Система автоматически помечает ответы флагом `is_suspicious = 1` если:

- Опрос пройден менее чем за **15 секунд**
- (В будущем: fingerprint-дубли, honeypot-поле заполнено)

**По умолчанию** они исключены из всей аналитики (фильтр «Exclude suspicious» включён).

Для их просмотра: снять галочку «Exclude suspicious» в фильтрах → в таблице Responses они отмечены ⚠.

**Рекомендация:** не удалять сразу — иногда быстрое прохождение легитимно (повторный ответ знакомого). Смотрите детали ответа перед удалением.

## 14. Бэкапы

Вкладка **Backups**.

### Автоматические бэкапы

Система автоматически создаёт бэкап:

- При каждом старте сервера
- Каждые 6 часов (настраивается)

Хранится последние 20 бэкапов, старые удаляются.

### Ручные бэкапы

- **Create Backup Now** — создать прямо сейчас (перед важными изменениями)
- **Download** — скачать файл `.db` себе
- **Upload Backup** — загрузить ранее скачанный файл

### Восстановление

Нажать **Restore** → подтверждение → система:

1. Автоматически создаёт бэкап текущего состояния (`pre-restore-...`)
2. Заменяет базу данных
3. Перезапускает соединение

> **Внимание:** Восстановление необратимо (кроме автоматического pre-restore бэкапа). Всегда создавайте ручной бэкап перед восстановлением.

## Частые вопросы

**Q: Почему средний скор изменился после смены фильтра?**

A: Все метрики пересчитываются по отфильтрованной выборке. Это нормально — разные группы имеют разный уровень выгорания.

**Q: Почему в Cross Analysis мало данных по некоторым группам?**

A: Небольшая выборка (n < 10) делает средние ненадёжными. Смотрите на столбец `n` в таблице.

**Q: Можно ли иметь несколько активных опросов?**

A: Нет. Одновременно активен только один опрос. Старый деактивируется автоматически при активации нового.

**Q: Как сравнить данные разных семестров?**

A: Создайте новый опрос для нового семестра (или деактивируйте текущий). Используйте фильтр по дате для разделения данных в аналитике.

**Q: Данные анонимны?**

A: Да. Хранится только хэш IP (не сам IP), fingerprint браузера (не идентифицирует личность), курс, институт, период семестра. Персональных данных не собирается.
